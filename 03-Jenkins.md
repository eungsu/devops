# 젠킨스

## Jenkins란?
- 젠킨스(Jenkins)는 자동화된 빌드, 테스트, 배포를 지원하는 지속적인 통합(Continuous Integration) 도구입니다. 소프트웨어 개발자들이 개발한 코드를 효율적으로 빌드하고, 테스트하며, 배포할 수 있도록 지원합니다.
- 젠킨스는 무료 오픈 소스 소프트웨어로, 다양한 빌드 환경을 지원하며, 다양한 플러그인을 제공하여 필요한 기능을 쉽게 추가할 수 있습니다. 또한 파이프라인을 구성하여 코드 변경 시 자동으로 빌드, 테스트, 배포를 수행할 수 있도록 지원하며, 이를 통해 애플리케이션의 개발 및 운영을 보다 효율적으로 처리할 수 있습니다.
- 젠킨스는 대규모 프로젝트에서도 사용이 가능하며, 사용자가 요구하는 대부분의 빌드 시나리오를 지원합니다. 또한 젠킨스는 자동화된 빌드, 테스트, 배포를 지원하여 개발자와 운영팀은 더욱 효율적으로 애플리케이션을 개발하고 배포할 수 있습니다.
- 다양한 플러그인과 기능을 지원하기 때문에, 젠킨스는 소프트웨어 개발, 인프라 자동화, 데이터 분석 등 다양한 분야에서 활용될 수 있습니다.

## Jenkins가 지원하는 다양한 작업
- 젠킨스는 다양한 작업을 수행할 수 있습니다. 일반적으로는 아래와 같은 작업을 수행합니다.
  * 소스 코드의 빌드 및 컴파일
  * 단위 테스트 및 코드 분석
  * 정적 코드 분석 및 보안 검사
  * 릴리스 및 배포 관리
  * 빌드 및 배포 로그 관리
  * 지속적인 통합 및 지속적인 배포 파이프라인 구축
  * Git, SVN 등과 같은 버전 관리 시스템과의 통합
  * AWS, GCP, Azure 등 클라우드 서비스와의 통합
  * JUnit, Selenium 등 다양한 테스트 도구와의 통합
  * Slack, HipChat 등 다양한 메시징 플랫폼과의 통합
- 위와 같은 작업을 자동화하여 젠킨스는 개발자들이 소프트웨어를 개발하고, 테스트하고, 릴리스하고, 배포하는데 필요한 번거로운 과정을 효율적으로 처리할 수 있도록 지원합니다. 이를 통해 개발 생산성을 높일 수 있습니다.

## Jenkins의 파이프라인
- 젠킨스의 파이프라인은 코드의 변경이 발생하면 자동으로 빌드, 테스트, 배포 등의 단계를 수행하는 자동화된 CI/CD 프로세스입니다.
- 파이프라인은 다양한 작업 단계를 정의하고 각 단계에서 수행되어야 할 작업을 지정할 수 있습니다. 예를 들어, 빌드, 테스트, 배포 단계로 구성된 파이프라인을 구성할 수 있습니다. 각 단계에서는 다양한 플러그인을 사용하여 필요한 작업을 수행할 수 있습니다. 예를 들어, 빌드 단계에서는 Maven, Gradle 등의 빌드 도구를 사용하며, 테스트 단계에서는 JUnit, Selenium 등의 테스트 도구를 사용할 수 있습니다.
- 또한 파이프라인은 다양한 조건에 따라 분기할 수 있습니다. 예를 들어, 특정 브랜치에 대해서만 빌드 및 테스트를 수행하거나, 코드 변경이 없는 경우 배포 단계를 건너뛸 수 있습니다.
- 젠킨스는 파이프라인을 GUI 기반으로 작성할 수 있으며, Jenkinsfile이라는 코드로 작성할 수도 있습니다. Jenkinsfile은 코드 저장소에 포함되어 있으며, 파이프라인을 구성하는 데 사용됩니다.
- 파이프라인을 사용하여 젠킨스는 더욱 자동화된 CI/CD 프로세스를 제공하며, 애플리케이션의 개발 및 운영을 보다 효율적으로 처리할 수 있도록 지원합니다.

## Jenkiks 설치하기
- 도커를 활용해서 젠킨스를 설치하는 방법은 아래와 같습니다.
  * 도커 이미지 다운로드 및 실행
  ```
  $ docker run -d -p 8080:8080 -p 50000:50000 jenkins/jenkins:lts
  ```
  > 위 명령어를 실행하여 jenkins/jenkins:lts 이미지를 다운로드하고 컨테이너를 실행합니다. `-d` 옵션은 컨테이너를 백그라운드로 실행하도록 지정합니다. `-p` 옵션은 호스트와 컨테이너의 포트를 매핑합니다.
  * 젠킨스 초기 설정
    + 웹 브라우저를 열어 `http://localhost:8080`으로 접속합니다. 초기 설정을 수행하며, 젠킨스 관리자 계정을 생성합니다.
  * 젠킨스 플러그인 설치
    + 젠킨스 초기 설정 후, 플러그인을 설치합니다. 젠킨스 관리자 페이지에서 "플러그인 관리" 메뉴를 선택합니다. 플러그인 목록에서 필요한 플러그인을 선택하고 설치를 진행합니다.
  * 젠킨스 설정 저장
    + 설치된 플러그인을 활성화하기 위해, 젠킨스 설정을 저장합니다. 이를 위해서는 "저장" 버튼을 클릭하면 됩니다.
- 이제 도커를 활용하여 젠킨스를 설치하고 설정하는 방법을 알아보았습니다. 이 방법을 통해, 개발자들은 간편하게 젠킨스를 구성하고, 소프트웨어를 지속적으로 통합하고 배포할 수 있습니다.
