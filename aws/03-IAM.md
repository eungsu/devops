#  IAM(Identity and Access Management)

## IAM이란?
- IAM은 **사용자의 접근 권한을 관리**하는 서비스다.
- IAM 을 통해서 회사 내 AWS를 사용하는 사람들에게 부서마다 사용자별로 AWS에서 제공하는 서비스들, 서비스에서 생성된 자원에 대해 세분된 권한을 지정할 수 있다.
- 대부분의 사용자 권한 관리 서비스와 마찬가지로, 최고 관리자가 Root 계정을 관리하고, 그밖의 사용자들은 각자 계정을 발급받아서 제한된 권한을 갖고 AWS을 이용하게 된다.
- IAM은 AWS 서비스에 대한 엑세스 권한 설정 뿐만 아니라, **언제 어디서 누가 무엇을 어떻게 할 수 있는가**을 상세히 설정할 수도 있다.
- IAM은 AWS 사용자 및 그룹을 관리하면 AWS 리소스에 대한 엑세스를 허용 및 거부 설정함으로써 전반적인 사용자 관리를 하는 서비스다.

## IAM의 주요 기능
- 사용자 생성/관리/계정의 보안 : 사용자의 패스워드 정책 관리
- AWS 계정에 대한 공유 엑세스 : AWS 계정의 리소스 관리 및 권한을 다른 사람에게 부여가능
- 세분화된 권한 : 리소스에 따라 여러 사람에게 다양한 권한 부여 가능
- EC2 애플리케이션 권한 자격 부여 : 사용자 뿐만 아니라, EC2에서 실행되는 애플리케이션에 IAM기능을 이용해 자격증명 제공이 가능하다.
- 멀티 팩터 인증(MFA) : 보안강화를 위해 암호나 엑세스 키뿐 아니라, 특별히 구성된 디바이스 코드를 제공하는 인증을 추가할 수 있다.
- 자격증명 연동 : 기업 네트워크, 인터넷 자격 증명 공급자 같이 다른 곳에 이미 암호가 있는 사용자에게 AWS 계정에 대한 임시 엑세스 권한 부여 가능
- 계정에 별명 부여 
- 글로벌 서비스 : AWS 계정은 전세계에서 유니크하다.

## IAM의 구성요소
- IAM은 AWS 클라우드 인프라 안에서 신분과 접속/접근을 관리하기 위한 서비스다.
- IAM은 크게 사용자(Users), 그룹(Groups), 역할(Roles), 정책(Policies)로 구성되어 있다.

### IAM 사용자(Users)
- 실제 AWS의 기능과 자원을 이용하는 **사람** 혹은 **애플리케이션**을 의미한다.
- 사용자(사람)가 AWS 서비스를 엑세스하는 것을 조정하는 것 뿐만 아니라, 애플리케이션이 서비스에 엑세스하는 것도 관리
- 아래와 같이 각 사용자마다 권한을 부여하는 방식이다.
  + 사용자1 : AdministratorAccess
  + 사용자2 : AmazonS3FullAccess
  + 사용자3 : AmazonS3ReadOnlyAccess AmazoneEC2ReadOnlyAccess

### IAM 그룹(Groups)
- 사용자가 적을 때는 직접 개별 사용자에게 권한을 설정하는 것이 더 직관적이고 편할 수 있지만, 사용자가 많아지면 관리하기가 불편해진다. 따라서 그룹 개념을 사용한다.
- 다수의 사용자를 모아놓은 개념이다.
- 하나의 **그룹에 여러 명으 사용자를 지정해서 공통적으로 권하는 주어야 하는 상황**일 때 유용하고, IAM 사용자마다 매번 정책을 직접 연결해줘야 하는 번거로움을 줄일 수 있다.
- 그룹은 동일한 권한을 가지는 사용자에게 권한을 설정하는 것을 편리하게 하는 위함이지, 그룹 자체가 보안 주체가 될 수는 없다.
- 하나의 사용자는 최대 10개의 그룹에 속할 수 있다.

### IAM 정책(Policy)
- 사용자, 그룹, 역할이 무엇을 할 수 있는지에 대한 **Permission 설정 모든 데이터 문서**
- 하나 이상의 AWS 리소스에 대해 어떤 작업을 수행할 수 있는지 허용 규칙을 **JSON 형식**으로 작성한다.
  + 어떤 사용자는 인스턴스를 올릴 수 있는데 삭제를 못하거나, 버킷에 접근으 가능한데 파일을 추가할 수 없거나 등
- 생성된 정책 문서를 IAM 사용자, 그룹, 역할에 연결한다.
- IAM 정책 타입 종류
  + 자격 증명 기반(Identity-based policies)
    + AWS 관리형 정책 - AWS에거 미리 제공하는 정책
    + AWS 고객 관리형 정책 - 고객이 직접 만들어서 사용하는 정책
    + AWS 인라인 정책 - 단일 사용자, 그룹, 역할에 직접 추가하는 방식 
  + 리소스 정책 기반(Resource-based policies) - EC2 같은 리소스에 적용하는 정책
  + 권한 경계 기반 정책(Permissions boundaries)
  + 조직 SCP 기반 정책(Organizations SCPs)
  + 엑세스 제어 리스트(Access control lists-ACLs)
  + 세션 정책(Session policies)
 
#### 자격증명 기반 정책(Id-Based Policy)
- 자격증명 기반 정책은 ID(사용자, 사용자 그룹, 역할)가 수행할 수 있는 작업, 리소스 및 조건을 제어하는 JSON 권한 정책 문서다.
- 자격증명 기반 정책은 3가지로 분류할 수 있다.
- AWS 관리형 정책
  + AWS 관리형 정책은 AWS에서 생성 및 관리하는 독립적인 정책이다.
  + IAM 탭에서 나오는 미리보기 정책들이 AWS 관리형 정책이다.
  + AWS에거 기본으로 제공하는 정책이다. 정책 스스로 정책이름이 포함된 Amazon 리소스 이름(ARN)을 갖고 있다.
- AWS 고객 관리형 정책
  + AWS 고객 관리형 정책은 **사용자가 정책을 새로 생성해서 커스텀**한 것을 말한다.
  + 고객 관리형 저책은 만들어진 Account에서만 사용할 수 있다.
- AWS 인라인 정책
  + AWS 인라인 정책은 **1 to 1 정책**으로 명시적으로 할당되는 정책이다.
  + 보통 IAM 정책은 다수의 사용자에게 여러 번 적용해서 사용되는데, 인라인 정책은 **하나의 사용자에게 하나의 정책을 적용**하는 것이다.
#### 리소스 기반 정책(Resource-based Policy)
- 자격증명 정책인 사용자나 그룹에 적용하는 정책이라면, 리소스 기반 정책은 AWS 리소스에 적용하는 정책이다.
- 대표적으로 S3 버킷 정책이 있다.
#### IAM policy JSON 문서 구조
- IAM policy는 JSON 파일로 구성되어 있으며, 여러 요소로 구성된다.
